# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

個人の学習記録サイト。[Docusaurus](https://docusaurus.io/) で構築し、GitHub Pages にデプロイされる。

## Commands

```bash
pnpm install          # 依存パッケージのインストール
pnpm start            # 開発サーバー起動（http://localhost:3000）
pnpm build            # 本番ビルド（./build に出力）
pnpm typecheck        # TypeScript 型チェック
pnpm lint             # ESLint 実行
pnpm format           # ESLint + Prettier による自動整形
```

ツールバージョン管理には mise を使用。`mise.toml` に Node.js 24 と pnpm 11 が定義されている。

## Architecture

### ドキュメント構成

`docs/` 配下のディレクトリ構造がそのままサイドバーに反映される（`autogenerated`）。

- ディレクトリ名は `NN-slug` 形式（例: `01-terasoluna`）で表示順を制御
- 各ディレクトリに `_category_.json` を置いてカテゴリ名・説明を定義
- `sidebars.ts` は `{ type: "autogenerated", dirName: "." }` のみ — 手動管理不要

### 設定ファイル

- `docusaurus.config.ts` — サイト全体の設定（URL、ナビゲーション、フッター等）
- `sidebars.ts` — サイドバー設定（自動生成に委ねているため編集不要）
- `eslint.config.ts` — ESLint flat config（TypeScript + React + import + Prettier）
- `tsconfig.json` — `@docusaurus/tsconfig` を継承、`NodeNext` モジュール解決

### デプロイ

- `main` ブランチへの push → `deploy.yml` が `gh-pages` ブランチへ自動デプロイ
- PR 作成時 → `test-deploy.yml` がビルド確認を実行
- どちらも `pnpm/action-setup@v4` → `actions/setup-node@v4` の順でセットアップ

## Adding Documentation

新しいトピックを追加するには `docs/` 配下に `NN-topic-name/` ディレクトリを作り、`_category_.json` と Markdown ファイルを置く。`NN` の数値で他カテゴリとの表示順を決める。
